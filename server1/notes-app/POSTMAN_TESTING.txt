================================================================================
                    POSTMAN TESTING GUIDE - NOTES API
================================================================================

================================================================================
PREREQUISITES
================================================================================

1. Install MongoDB locally or use MongoDB Atlas
2. Install Node.js and npm
3. Install Postman
4. Start your server: npm start
5. Import this collection into Postman

================================================================================
BASE URL
================================================================================

http://localhost:3000/api/notes

================================================================================
ENDPOINT 1: CREATE A NOTE
================================================================================

URL: http://localhost:3000/api/notes
Method: POST
Headers:
  - Key: Content-Type
    Value: application/json

Body (JSON):
{
  "title": "My First Note",
  "content": "This is the content of my first note",
  "category": "personal",
  "isPinned": false
}

Test Cases:
-----------
✓ Valid note creation
✓ Missing title (should return 400)
✓ Missing content (should return 400)
✓ All fields provided

Expected Response:
{
  "success": true,
  "message": "Note created successfully",
  "data": {
    "title": "My First Note",
    "content": "This is the content of my first note",
    "category": "personal",
    "isPinned": false,
    "_id": "...",
    "createdAt": "...",
    "updatedAt": "..."
  }
}

================================================================================
ENDPOINT 2: CREATE MULTIPLE NOTES (BULK)
================================================================================

URL: http://localhost:3000/api/notes/bulk
Method: POST
Headers:
  - Key: Content-Type
    Value: application/json

Body (JSON):
{
  "notes": [
    {
      "title": "Work Note 1",
      "content": "Work content 1",
      "category": "work"
    },
    {
      "title": "Study Note 1",
      "content": "Study content 1",
      "category": "study"
    },
    {
      "title": "Personal Note 1",
      "content": "Personal content 1",
      "category": "personal",
      "isPinned": true
    }
  ]
}

Test Cases:
-----------
✓ Valid bulk creation (3 notes)
✓ Empty notes array (should return 400)
✓ Missing notes field (should return 400)

Expected Response:
{
  "success": true,
  "message": "3 notes created successfully",
  "data": [
    { "_id": "...", "title": "Work Note 1", ... },
    { "_id": "...", "title": "Study Note 1", ... },
    { "_id": "...", "title": "Personal Note 1", ... }
  ]
}

================================================================================
STEP-BY-STEP TESTING WORKFLOW
================================================================================

STEP 1: Create a Note
---------------------
1. Select POST method
2. Enter: http://localhost:3000/api/notes
3. Go to Headers tab, add Content-Type: application/json
4. Go to Body tab, select "raw" and "JSON"
5. Paste: {"title": "My Note", "content": "Note content", "category": "work"}
6. Click Send
7. Copy the "_id" from response for next steps

STEP 2: Create Bulk Notes
--------------------------
1. Select POST method
2. Enter: http://localhost:3000/api/notes/bulk
3. Add Header: Content-Type: application/json
4. Body:
{
  "notes": [
    {"title": "Work 1", "content": "Work content", "category": "work"},
    {"title": "Study 1", "content": "Study content", "category": "study"}
  ]
}
5. Click Send

STEP 3: Get All Notes
---------------------
1. Select GET method
2. Enter: http://localhost:3000/api/notes
3. Click Send
4. Note down some note IDs

STEP 4: Get Single Note
------------------------
1. Select GET method
2. Enter: http://localhost:3000/api/notes/{COPIED_ID}
3. Click Send

STEP 5: Update Note (PUT)
--------------------------
1. Select PUT method
2. Enter: http://localhost:3000/api/notes/{COPIED_ID}
3. Add Header: Content-Type: application/json
4. Body (ALL fields required):
{
  "title": "Updated Title",
  "content": "Updated content",
  "category": "study",
  "isPinned": true
}
5. Click Send

STEP 6: Patch Note (PATCH)
---------------------------
1. Select PATCH method
2. Enter: http://localhost:3000/api/notes/{COPIED_ID}
3. Add Header: Content-Type: application/json
4. Body (only fields to update):
{"title": "Partially Updated"}
5. Click Send

STEP 7: Delete Note
--------------------
1. Select DELETE method
2. Enter: http://localhost:3000/api/notes/{COPIED_ID}
3. Click Send

STEP 8: Bulk Delete
-------------------
1. Select DELETE method
2. Enter: http://localhost:3000/api/notes/bulk
3. Add Header: Content-Type: application/json
4. Body:
{"ids": ["ID1", "ID2", "ID3"]}
5. Click Send

================================================================================
ERROR TESTING
================================================================================

Test Invalid ObjectId:
----------------------
URL: http://localhost:3000/api/notes/invalid123
Method: GET

Expected: 400 Bad Request
{
  "success": false,
  "message": "Invalid note ID"
}

Test Non-Existent ID:
---------------------
URL: http://localhost:3000/api/notes/507f1f77bcf86cd799439011
Method: GET

Expected: 404 Not Found
{
  "success": false,
  "message": "Note not found"
}

Test Missing Required Fields:
------------------------------
URL: http://localhost:3000/api/notes
Method: POST
Body: {"title": "Only Title"}

Expected: 400 Bad Request
{
  "success": false,
  "message": "Title and content are required"
}

Test Empty PATCH Body:
-----------------------
URL: http://localhost:3000/api/notes/{VALID_ID}
Method: PATCH
Body: {}

Expected: 400 Bad Request
{
  "success": false,
  "message": "Request body cannot be empty"
}

Test Invalid Category:
-----------------------
URL: http://localhost:3000/api/notes
Method: POST
Body: {"title": "Test", "content": "Content", "category": "invalid"}

Expected: 400 Bad Request (validation error)

================================================================================
TIPS FOR BEGINNERS
================================================================================

1. Always set Content-Type header to application/json for POST/PUT/PATCH/DELETE
2. Use the Body tab in Postman and select "raw" then "JSON"
3. Copy note IDs from responses to use in subsequent requests
4. Check the "success" field in response to know if operation succeeded
5. Use the console in Postman to debug requests
6. Test error cases to ensure proper validation
