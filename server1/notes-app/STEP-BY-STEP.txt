STEP 6: CONTROLLER (BUSINESS LOGIC)
================================================================================

6.1 Created src/controllers/note.controller.js

This is where ALL the logic resides:

CREATE NOTE (createNote):
- Extracts title, content, category, isPinned from request body
- Validates title and content are provided
- Uses Note.create() to insert document
- Returns 201 with created note

CREATE BULK NOTES (createBulkNotes):
- Expects { notes: [...] } array
- Validates notes array exists and is not empty
- Maps array to Note objects with defaults
- Uses Note.insertMany() for bulk insert
- Returns 201 with all created notes

GET ALL NOTES (getAllNotes):
- Uses Note.find() with no filters
- Returns array of all notes

GET NOTE BY ID (getNoteById):
- Validates ObjectId format
- Uses Note.findById(id)
- Returns 404 if not found
- Returns note with 200

UPDATE NOTE (updateNote):
- PUT method - full replacement
- Uses { new: true, overwrite: true, runValidators: true }
- All fields required in request body
- Validates all fields including category enum

PATCH NOTE (patchNote):
- PATCH method - partial update
- Uses { new: true, runValidators: true }
- Only updates provided fields
- Fails if body is empty
- Flexible field updates

DELETE NOTE (deleteNote):
- Uses Note.findByIdAndDelete(id)
- Returns 404 if not found
- Returns deleted note in response

DELETE BULK NOTES (deleteBulkNotes):
- Expects { ids: [...] } array
- Validates IDs array
- Filters to valid ObjectIds only
- Uses Note.deleteMany() with $in operator
- Returns count of deleted notes

KEY DIFFERENCES:

PUT vs PATCH:
- PUT: { new: true, overwrite: true, runValidators: true }
  * Replaces entire document
  * Missing fields = undefined/null
  * Use when you want full replacement

- PATCH: { new: true, runValidators: true }
  * Updates only provided fields
  * Other fields remain unchanged
  * Use for partial updates

BULK OPERATIONS:
- insertMany(): Fast bulk insert, returns array
- deleteMany(): Deletes multiple documents matching criteria

================================================================================
KEY DIFFERENCES: PUT vs PATCH
================================================================================

PUT /api/notes/:id
- Full replacement
- All fields required
- Missing fields become undefined
- Use when: Client sends complete resource

PATCH /api/notes/:id
- Partial update
- Only changed fields required
- Other fields unchanged
- Use when: Client sends only changes

Example:
Original: { title: "A", content: "B", category: "work", isPinned: false }

PUT with { title: "X" }:
Result: { title: "X", content: undefined, category: undefined, isPinned: undefined }

PATCH with { title: "X" }:
Result: { title: "X", content: "B", category: "work", isPinned: false }

================================================================================
MONGOOSE METHODS USED
================================================================================

Note.create():
- Creates single document
- Validates against schema
- Returns created document

Note.insertMany():
- Creates multiple documents
- Faster than multiple create()
- Atomic operation
- Returns array of created documents

Note.find():
- Returns all documents matching query
- Empty query = all documents

Note.findById(id):
- Finds single document by _id
- Returns null if not found

Note.findByIdAndUpdate(id, updates, options):
- findByIdAndUpdate with:
  * new: true → return updated document
  * overwrite: true → full replacement (PUT)
  * runValidators: true → validate before update

Note.findByIdAndDelete(id):
- Finds and deletes document
- Returns deleted document
- Returns null if not found

Note.deleteMany(filter):
- Deletes all documents matching filter
- Uses $in operator for multiple IDs
- Returns { deletedCount }

COMMON ISSUES & SOLUTIONS
================================================================================

Issue: MongoDB connection refused
Solution: Ensure MongoDB is running (mongod)

Issue: Validation error on category
Solution: Use only: work, personal, study

Issue: Invalid ObjectId format
Solution: Use 24-character hex string

Issue: PUT clears fields
Solution: This is expected behavior for PUT (full replacement)

Issue: PATCH requires body
Solution: Include at least one field to update

Issue: Bulk operations fail
Solution: Ensure proper array format in request body
